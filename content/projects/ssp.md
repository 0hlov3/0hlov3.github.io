> Cloud-based self-service platform enabling automated deployment of healthcare applications on Kubernetes.

Deployment and operation of a **self-service platform** for healthcare applications based on Azure Kubernetes Service (AKS), enabling automated provisioning of services following customer orders.

The platform was designed to provide a scalable and standardized environment for deploying applications in a cloud-native architecture. Close collaboration with development teams ensured smooth integration of application delivery workflows into the platform.

Infrastructure and platform components were provisioned and managed using Infrastructure as Code (Terraform), while deployment processes were fully automated using CI/CD pipelines and Helm-based release workflows.

A strong focus was placed on automation, reproducibility, and operational efficiency, enabling consistent deployments across environments and reducing manual intervention.

## Key Responsibilities

- Deployment and operation of Kubernetes clusters on Azure Kubernetes Service (AKS)
- Infrastructure provisioning using Terraform
- Implementation and maintenance of CI/CD pipelines (Azure DevOps, Azure Pipelines)
- Helm-based application deployment and release management
- Collaboration with development teams to integrate application workflows into the platform
- Optimization of deployment processes and platform operations
- Troubleshooting and support for application and platform issues

## Technologies & Methods

- Azure Kubernetes Service (AKS)
- Terraform (Infrastructure as Code)
- Azure DevOps & Azure Pipelines
- Helm (application deployment)
- CI/CD automation
- Cloud-native architecture
- Platform engineering practices

## Impact

- Enabled self-service provisioning of applications, reducing manual deployment effort
- Established standardized and reproducible deployment workflows
- Improved delivery speed and operational efficiency for application teams
- Provided a scalable cloud platform for healthcare-related services
